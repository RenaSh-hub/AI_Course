Feature: DS-131 — Fix duplicate name rejection on edit (DS-2-TC-010)

  As an admin, when I edit a program I want duplicate Program Names rejected
  so that two active programs cannot share the same name within the organization.

  Tracks fix for bug DS-126 (Edit Program accepts duplicate name on save).
  Parent story: DS-2 — Edit existing program details.
  Primary AC reference: DS-2 Scenario "Duplicate active program name is rejected on Save"
  and Validation Rules (duplicate → 400/409 with visible error).

  # Regression — happy path still works

  Scenario: TC-001 — Valid rename to a unique name still succeeds
    Given I am logged in as admin
    And I am on the Programs page
    And a program "Web Development 2026 <suffix>" exists
    When I open Edit for that program
    And I change the Program Name to a unique name
    And I click Save
    Then the Edit Program modal closes
    And the program list shows the unique new name

  # Core fix — duplicate rejection on edit

  Scenario: TC-002 — Duplicate name on edit is rejected with visible error
    Given I am logged in as admin
    And active programs "Program A <suffix>" and "Program B <suffix>" exist
    When I open Edit for "Program A <suffix>"
    And I change the Program Name to "Program B <suffix>" exactly
    And I click Save
    Then a visible duplicate-name error is shown
    And the Edit Program modal remains open

  Scenario: TC-003 — Both programs retain original names after rejected edit
    Given I am logged in as admin
    And active programs "Program A <suffix>" and "Program B <suffix>" exist
    When I attempt to rename "Program A <suffix>" to "Program B <suffix>" and Save
    Then the program list still shows "Program A <suffix>"
    And the program list still shows "Program B <suffix>"

  Scenario: TC-004 — Program count unchanged after rejected duplicate rename
    Given I am logged in as admin
    And active programs "Program A <suffix>" and "Program B <suffix>" exist
    When I attempt to rename "Program A <suffix>" to "Program B <suffix>" and Save
    Then "Program A <suffix>" remains visible in the list
    And the original program is not removed from the list by the rejected save

  # Variants from Validation Rules / DS-2 ACs

  Scenario: TC-005 — Case-insensitive duplicate on edit is rejected
    Given I am logged in as admin
    And active programs "Program A <suffix>" and "Program B <suffix>" exist
    When I open Edit for "Program A <suffix>"
    And I change the Program Name to "Program B <suffix>" in UPPERCASE
    And I click Save
    Then a visible duplicate-name error is shown
    And the Edit Program modal remains open
    And the program list still shows "Program A <suffix>"

  Scenario: TC-006 — Whitespace-wrapped duplicate on edit is rejected
    Given I am logged in as admin
    And active programs "Program A <suffix>" and "Program B <suffix>" exist
    When I open Edit for "Program A <suffix>"
    And I change the Program Name to "  Program B <suffix>  "
    And I click Save
    Then the update is rejected (modal stays open and/or duplicate error shown)
    And the program list still shows "Program A <suffix>"
