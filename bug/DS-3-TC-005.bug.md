## Summary

Three Playwright tests in `tests/ds3-name-validation.spec.ts` fail on CI: creating a second
program with an existing name succeeds. The app does not enforce server-side
(case/trim-insensitive) uniqueness on Program Name, so no duplicate error is shown and a
second row is added.

## Affected tests

| TC | Location | Symptom |
| --- | --- | --- |
| TC-005 | `tests/ds3-name-validation.spec.ts:66` | No `already exists/duplicate` error in dialog |
| TC-008 | `tests/ds3-name-validation.spec.ts:122` | `programRow(name)` resolves to 2 (expected 1) |
| TC-009 | `tests/ds3-name-validation.spec.ts:146` | Trimmed `"  name  "` accepted, no error |

## Steps to reproduce

1. Create a program named `Dup Check <ts>`
2. Open **+ New Program** again, enter the same name
3. Click **Create**

## Expected result

- Save rejected; dialog shows error matching `/already exists|duplicate/i` (DS-3 AC: "Reject duplicate program name")
- List contains exactly 1 row with that name

## Actual result

- No duplicate error appears in the New Program dialog (TC-005, TC-009)
- A second row is created → `programRow(name)` resolves to 2 (TC-008)
- Trimmed variant `"  name  "` also accepted (TC-009)
