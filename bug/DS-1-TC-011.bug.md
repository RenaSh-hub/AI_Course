# DS-92 — [Rena] Create Program: double-click on Create button produces duplicate program (Playwright TC-011)

**Jira:** [DS-92](https://legionqaschool.atlassian.net/browse/DS-92)

**Spec:** `tests/pw-ds1.spec.ts`
**Priority:** High

---

## Summary

Playwright **TC-011** in `tests/pw-ds1.spec.ts` fails: double-clicking the **Create** button on the **New Program** modal submits the form twice, creating **two identical programs** instead of one.

## Preconditions

- Admin user signed in at https://test.didaxis.studio
- On the **Programs** page

## Steps to reproduce

1. Sign in as **Admin** at https://test.didaxis.studio
2. Navigate to **Programs**
3. Click **+ New Program**
4. Fill in **Program Name** with `Mobile Development 2026`
5. Fill in **Description** with `iOS and Android development`
6. **Double-click** the **Create** button

## Expected result

- Modal closes
- Exactly **one** program named `Mobile Development 2026` appears in the program list
- The Create button should be disabled after the first click or the form should have idempotency protection

## Actual result

- Modal closes
- **Two** programs named `Mobile Development 2026` appear in the program list
- No duplicate prevention or button-disable guard on submit

## Error output

```
expect(locator).toHaveCount(expected)

Expected: 1
Received: 2
```
