## Summary

Playwright **TC-018** in `tests/ds4-delete-program.spec.ts` fails on CI after a seeded assertion change for **Block 10** testing. The test expects the program row to **disappear** after the native delete confirmation dialog is **dismissed**, but the app **keeps the program in the list** — which matches DS-4 acceptance criteria (dismiss = cancel).

## Preconditions

- Admin user signed in at https://test.didaxis.studio
- User is on **Programs** (`/programs`)
- A disposable program exists in the list (created via **+ New Program**)

## Steps to reproduce

1. Sign in as **Admin** at https://test.didaxis.studio
2. Go to **Programs**
3. Create a program named e.g. `Del Esc {timestamp}`
4. Click **Delete {program name}** on the program row
5. When the native confirm dialog appears, **dismiss** it (Playwright: `dialog.dismiss()`)
6. Observe the program list

## Expected result (per DS-4 test plan TC-018)

- Dismiss/Esc behaves the same as **Cancel**
- No delete occurs; program row **remains visible** in the list
- Playwright assertion should be: `await expect(row).toBeVisible()`

## Actual result (CI failure)

- Program row **remains visible** (correct product behaviour)
- Playwright assertion `await expect(row).not.toBeVisible()` **fails** after 5s timeout
