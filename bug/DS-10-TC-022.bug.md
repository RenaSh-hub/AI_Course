## Classification

| Field | Value |
| --- | --- |
| **Type** | Real application defect |
| **Feature** | Create Program — New Program modal |
| **Severity** | Medium — no data saved on cancel, but stale draft reappears and **Create** stays enabled, risking accidental submit |
| **Root cause** | Modal form state is not reset when dismissed via Cancel or close X; reopening reuses prior field values |

## Summary

After filling **Program Name** and **Description** in the New Program modal and dismissing
without saving (Cancel or close X), clicking **+ New Program** again shows the previous
draft values. **Create** remains enabled. Existing DS-1 specs (TC-007, TC-007b) only assert
that no program row appears — they do not check form reset.

## Steps to reproduce

1. Sign in as **Admin** at https://test.didaxis.studio
2. Go to **Programs**
3. Click **+ New Program**
4. Fill **Program Name** with `Draft Program Alpha`
5. Fill **Description** with `Draft description to discard`
6. Click **Cancel**
7. Click **+ New Program** again

## Expected result

- **Program Name** textbox is empty
- **Description** textbox is empty
- **Create** button is disabled

## Actual result

- **Program Name** still contains `Draft Program Alpha`
- **Description** still contains `Draft description to discard`
- **Create** button is enabled

Same behavior when dismissing via the dialog close X (TC-023).
