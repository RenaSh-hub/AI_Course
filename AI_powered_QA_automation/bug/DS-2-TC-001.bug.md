# DS-93 — [Rena] Edit Program: Edit button (✏️) not found or clickable in Programs list row (Playwright TC-001)

**Jira:** [DS-93](https://legionqaschool.atlassian.net/browse/DS-93)
**Parent story:** DS-2 — Edit existing program details
**Spec:** `tests/pw-ds2.spec.ts`
**Priority:** High

---

## Summary

Playwright **PW-DS-2** suite (`tests/pw-ds2.spec.ts`) fails on **all 24 tests** at `openEditModal()`: after a program is created and appears in the list, automation cannot find or click the row **Edit** control with accessible name `✏️`. The failure blocks validation of DS-2 edit flows (happy paths, negative cases, and edge cases).

## Preconditions

- Admin user signed in at https://test.didaxis.studio
- User is on **Programs** (`/programs`)
- At least one program exists in the list (created via **+ New Program**)

## Steps to reproduce

1. Sign in as **admin@didaxis.studio** at https://test.didaxis.studio
2. Go to **Programs**
3. Click **+ New Program**, enter a unique name (e.g. `Web Development 2026 1779751818892`) and description, click **Create**
4. Confirm the new program name is visible in the table
5. Locate the row for that program and click the **Edit** control (pencil / ✏️)

## Expected result

- The row exposes a discoverable **Edit** action (button or equivalent) with a stable accessible name (e.g. `Edit` or `✏️`)
- Clicking it opens the **Edit Program** dialog
- Playwright locator succeeds:
  `getByRole('row').filter({ hasText: '<programName>' }).getByRole('button', { name: '✏️' })`

## Actual result

- Program creation succeeds; the program name is visible in the list
- Playwright waits 60s for `button` named `✏️` inside the matching row, then times out
- **Edit Program** modal never opens
- Entire `pw-ds2.spec.ts` suite fails (24/24) at the same step

## Error output

```
locator.click: Test timeout of 60000ms exceeded.
Call log:
  - waiting for getByRole('row').filter({ hasText: 'Web Development 2026 1779751818892' }).getByRole('button', { name: '✏️' })
```

Location: `openEditModal()` — `tests/pw-ds2.spec.ts:27`

## Test reference

- Trace: `test-results/pw-ds2-PW-DS-2-Happy-paths-TC-001-—-Open-program-for-editing-chromium/trace.zip`
- Same failure pattern in TC-002 through TC-024

## Likely causes (for investigation)

1. Edit control only visible on **row hover** (not exposed until hover)
2. Accessible name mismatch — UI may use `Edit` / `aria-label` instead of emoji `✏️`
3. Edit control is not exposed with `role="button"` (icon link/span without button semantics)
